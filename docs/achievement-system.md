# 成就系统

## 概述

成就在特定时机检测条件，达成后持久化到 `localStorage`，跨局保留。

## 数据结构 (achievement.json)

```json
{
  "2001": {
    "id": 2001,
    "name": "初恋萌芽",
    "description": "心动值首次达到5",
    "grade": 0,
    "condition": "HLOV>=5",
    "hide": 0,
    "opportunity": "TRAJECTORY"
  }
}
```

| 字段 | 说明 |
|------|------|
| `grade` | 稀有度等级（0-3） |
| `condition` | 达成条件表达式 |
| `hide` | 是否为隐藏成就（未达成时不显示描述） |
| `opportunity` | 检测时机 |

## 检测时机 (Opportunity)

| 时机 | 触发点 | 说明 |
|------|--------|------|
| `START` | 点击"开始新学期"后 | 检测重开次数、成就数量等跨局数据 |
| `TRAJECTORY` | 每学期结束后 | 检测当前局内属性 |
| `SUMMARY` | 点击"恋爱总结"后 | 检测最终属性、总评等 |
| `END` | 点击"重开"后 | 重开次数 +1 后检测 |

## 检测流程

`Achievement.achieve(opportunity)`:

1. 列出所有成就
2. 过滤已达成的
3. 过滤当前时机不匹配的
4. 对剩余成就检查 condition
5. 达成的记录到 `localStorage` 并触发 UI 弹窗

## 成就展示

`Life.achievements` getter 返回排序后的成就列表：
- 已达成的按时间倒序
- 未达成且非隐藏的按等级排序
- 隐藏且未达成的排最后

## 统计数据

| 属性 | 说明 |
|------|------|
| `CACHV` | 已达成成就数量 |
| `TACHV` | 成就总数 |
| `RACHV` | 成就达成率 |
