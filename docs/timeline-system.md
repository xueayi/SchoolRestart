# 时间线系统

## 概述

游戏以**学期**（SEM）为最小时间单位推进，共 101 个学期，覆盖从出生到大学毕业的完整校园时间线。

## 阶段划分

| 阶段 | 学期范围 | 回合数 | 说明 |
|------|---------|--------|------|
| 婴幼儿 | 1-3 | 3 | 出生、一岁、两岁 |
| 幼儿园 | 4-9 | 6 | 小班上/下、中班上/下、大班上/下 |
| 小学 | 10-27 | 18 | 一年级到六年级，每年3学期（上/下/暑假） |
| 初中 | 28-45 | 18 | 初一到初三，每年6个月节点 |
| 高中 | 46-69 | 24 | 高一到高三，每年8个月节点 |
| 大学 | 70-101 | 32 | 大一到大四，每年8个月节点 |

## SEM_LABELS

定义在 `src/ui/themes/*/trajectory.js` 的静态数组中，将学期数字映射为人类可读的标签：

```javascript
static SEM_LABELS = [
    '',
    '出生', '一岁', '两岁',
    '幼儿园小班上', '幼儿园小班下', ...
    '小一·上', '小一·下', '小一·暑假', ...
    '初一·九月', '初一·十月', ...
    '高一·九月', ...
    '大四·毕业',
];
```

两个主题 (default/cyber) 维护相同的标签数组。

## 数据结构 (age.json)

每个学期在 `age.json` 中对应一条记录：

```json
{
  "28": {
    "age": 28,
    "event": ["15000", "15001", ...],
    "guaranteed": ["30009", "", "", "", ""],
    "talent": []
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `event` | `string[]` | 该学期的随机事件池（事件 ID 列表） |
| `guaranteed` | `string[]` | 该学期的保底事件（必定触发） |
| `talent` | `string[]` | 该学期解锁的天赋 ID |

## 推进机制

`Property.ageNext()` 每次调用：

1. `SEM += 1`
2. 从 `age.json` 查询当前学期的 `event`、`talent`、`guaranteed`
3. 返回给 `Life.next()` 处理

## 结束条件

`Property.isEnd()` 检查 `SEM >= maxSem`（默认 101），到达后游戏进入总结阶段。
